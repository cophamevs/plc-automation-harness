# S7-1500 Specific Features

## Frontmatter
- **Tags**: 1500, variant, oop, lreal, lint, optimized, technology
- **CPU**: 1500
- **TIA Version**: Any

## Overview
Features available ONLY on S7-1500 (not on S7-1200). Read this when targeting
S7-1500 to take advantage of its full capabilities.

---

## 1. Optimized vs Standard Block Access

| Mode | S7_Optimized_Access | Memory Layout | External Access |
|------|---------------------|---------------|-----------------|
| Optimized | TRUE (default) | Compiler-optimized, no fixed offsets | Symbolic only |
| Standard | FALSE | Sequential, fixed byte offsets | Absolute (DB1.DBW0) + Symbolic |

**When to use Standard (FALSE):**
- When reading via S7.Net (S7ReadDB, S7ReadDBStruct) — requires known byte offsets
- When exchanging data with HMI via absolute addresses
- When using PUT/GET communication

**When to use Optimized (TRUE):**
- Internal data that only PLC code accesses
- Better memory utilization and performance
- Default for new blocks — leave it unless you need absolute access

```scl
// Set in DB declaration:
DATA_BLOCK "DB_ForS7Net"
{ S7_Optimized_Access := 'FALSE' }   // Standard: fixed offsets
VERSION : 0.1
  // ...
END_DATA_BLOCK
```

## 2. 64-Bit Types

| Type | Size | Use Case |
|------|------|----------|
| LINT | 8 bytes | Large counters, timestamps, high-precision integers |
| ULINT | 8 bytes | Unsigned large values, bit masks |
| LREAL | 8 bytes | High-precision floating point (scientific, financial) |
| LWORD | 8 bytes | 64-bit raw data, packed bits |
| LTIME | 8 bytes | Nanosecond-precision time intervals |
| LDT | 8 bytes | Date-time with nanosecond precision |

```scl
VAR
  preciseTime  : LTIME;
  bigCounter   : LINT;
  preciseValue : LREAL;
END_VAR

#preciseTime := LT#0ns;
#bigCounter := LINT#0;
#preciseValue := LREAL#3.141592653589793;
```

## 3. VARIANT Type

Generic pointer that can reference any data type at runtime.

```scl
FUNCTION_BLOCK "FB_GenericLogger"
VERSION : 0.1

VAR_INPUT
  DataSource : VARIANT;   // Can point to ANY type
END_VAR

VAR_TEMP
  dataType : INT;
  intVal   : INT;
  realVal  : REAL;
END_VAR

BEGIN
  // Check what type VARIANT points to
  #dataType := TypeOf(#DataSource);
  
  CASE #dataType OF
    1:  // BOOL
      ;
    5:  // INT
      MOVE_BLK_VARIANT(SRC := #DataSource,
                        COUNT := 1,
                        SRC_INDEX := 0,
                        DEST_INDEX := 0,
                        DEST := #intVal);
    8:  // REAL
      MOVE_BLK_VARIANT(SRC := #DataSource,
                        COUNT := 1,
                        SRC_INDEX := 0,
                        DEST_INDEX := 0,
                        DEST := #realVal);
  END_CASE;
END_FUNCTION_BLOCK
```

**TypeOf() return values:**
| Value | Type |
|-------|------|
| 0 | VOID |
| 1 | BOOL |
| 2 | BYTE |
| 3 | WORD |
| 4 | DWORD |
| 5 | INT |
| 6 | DINT |
| 7 | SINT |
| 8 | REAL |
| 14 | STRING |
| 19 | ARRAY |
| 22 | STRUCT |

## 4. OOP Features

### METHOD
```scl
FUNCTION_BLOCK "FB_Motor"
VERSION : 0.1

VAR
  _speed : REAL;
  _running : BOOL;
END_VAR

METHOD PUBLIC "Start"
  #_running := TRUE;
END_METHOD

METHOD PUBLIC "Stop"
  #_running := FALSE;
  #_speed := 0.0;
END_METHOD

METHOD PUBLIC "SetSpeed"
VAR_INPUT
  speed : REAL;
END_VAR
  IF #_running THEN
    #_speed := #speed;
  END_IF;
END_METHOD

METHOD PUBLIC "GetSpeed" : REAL
  #GetSpeed := #_speed;
END_METHOD
```

### PROPERTY
```scl
// Inside FB declaration:
PROPERTY PUBLIC "Speed" : REAL

// GET accessor
GET
  #Speed := #_speed;
END_GET

// SET accessor
SET
  IF #Speed >= 0.0 AND #Speed <= 3000.0 THEN
    #_speed := #Speed;
  END_IF;
END_SET
```

### INTERFACE (S7-1500 V2.0+)
```scl
INTERFACE "IControllable"
  METHOD "Start";
  METHOD "Stop";
  METHOD "GetState" : INT;
END_INTERFACE
```

**OOP Guidelines:**
- Use sparingly — most PLC programmers expect procedural style
- Methods are good for encapsulating complex initialization or reset sequences
- Properties are good for validated setpoints
- Avoid deep inheritance hierarchies — keep it to 1-2 levels max
- INTERFACE is useful for swappable implementations (e.g., mock vs real sensor)

## 5. System Diagnostics

### Read Diagnostic Data
```scl
VAR_TEMP
  diagData : ARRAY[0..3] OF BYTE;
  retVal   : INT;
END_VAR

#retVal := DPRD_DAT(LADDR := 256,     // HW identifier of module
                     RECORD := #diagData);
// retVal: 0 = OK, 16#8x = error
```

### LED Control
```scl
// Read LED state
#ledState := LED(LADDR := 0, LED := 2);
// LED 2 = SF (system fault), returns: 0=off, 1=on, 2=blink
```

## Key Points
- Default to Optimized access unless you need S7.Net/HMI absolute access
- VARIANT is powerful but requires TypeOf() checks — don't overuse
- OOP features are S7-1500 only — code will NOT compile on S7-1200
- 64-bit types give precision but use 2x memory — only use when needed
- Max call nesting: 24 levels (vs 6 on S7-1200)
